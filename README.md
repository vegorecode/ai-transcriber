# WhisperX Transcription Docker Setup

Этот проект предоставляет настройку на базе Docker для транскрипции аудиофайлов с использованием [WhisperX](https://github.com/m-bain/whisperX). Он поддерживает автоматическое распознавание речи (ASR), обнаружение голосовой активности (VAD), диаризацию спикеров и вывод в форматах вроде SRT. Настройка включает предобработку аудио (конвертацию в 16kHz моно WAV с нормализацией шума).

Тестировалось с моделью `large-v3`, с попыткой использовать дообученную русскую модель (`antony66/whisper-large-v3-russian`), но столкнулся с багами архитектуры (см. [открытые issues на GitHub](https://github.com/m-bain/whisperX/issues) – разработчики обещают фиксы).
*В дальнейшем соберу, если будет нужда, от этих чертил не дождешься.

---

## Требования

* Установленные Docker и Docker Compose.

* NVIDIA GPU с поддержкой CUDA (настройка резервирует все доступные GPU).

* Аккаунт на Hugging Face для доступа к моделям (требуется для PyAnnote).

## Настройка

1.  **Клонируйте репозиторий** (или создайте файлы на основе предоставленных `docker-compose.yml` и `run.sh`).

2.  **Создайте файл `.env`** в корне проекта:

    * **Как получить HF_TOKEN**:

        * Зарегистрируйтесь или войдите в [Hugging Face](https://huggingface.co/).

        * Примите условия: 
        https://huggingface.co/pyannote/segmentation-3.0
        https://huggingface.co/pyannote/speaker-diarization
        https://huggingface.co/pyannote/speaker-diarization-3.1 
        

        * Перейдите в настройки профиля > Access Tokens.

        * Создайте новый токен с доступом "Read".

        * Скопируйте токен (начинается с `hf_`) и вставьте в `.env`.

        * Этот токен требуется для скачивания gated-моделей (например, PyAnnote для диаризации) и избежания лимитов.

3.  **Подготовьте входное аудио**:

    * Поместите аудиофайлы в директорию `./audio` (поддерживает MP3, WAV, OGG, M4A, FLAC, WEBM, MP4, MKV).

    * Скрипт автоматически конвертирует их в 16kHz моно WAV с нормализацией громкости (`loudnorm=I=-16:TP=-1.5:LRA=11` через FFmpeg) для оптимальной работы Whisper.

4.  **Настройте параметры** (опционально):

    * Измените переменные окружения в `docker-compose.yml` под свои нужды (см. раздел Конфигурация ниже).

    * Для русского аудио (как в примере) установите `LANGUAGE=ru` и `MODEL=large-v3` (или попробуйте кастомные модели после фиксов багов).

## Использование

Запустите транскрипцию командой:

```
docker compose run --rm whisperx
```

* **Вход**: Файлы в `./audio` (монтируется как read-only).

* **Выход**: Транскрипты в `./out` (например, SRT-файлы с таймстампами и метками спикеров).

* Контейнер обработает все файлы последовательно, сначала конвертируя при необходимости.

* Кэш сохраняется в volume `./whisper_cache` для ускорения повторных запусков.

Пример вывода (из предоставленного SRT):

* Транскрипт с диаризацией спикеров и таймингами, например:

```
1
00:00:00,031 --> 00:00:03,816
[SPEAKER_00]: Здравствуйте, Сергей.
```

Если аудиофайлы не найдены, скрипт выйдет с ошибкой.

## Конфигурация

Переменные окружения в `docker-compose.yml` управляют поведением. По умолчанию настроены для русских интервью (1-4 спикера). Настройте их под свое аудио:

| Переменная | Дефолт | Описание | Когда менять |
| :--- | :--- | :--- | :--- |
| `HF_TOKEN` (из .env) | | Токен Hugging Face для доступа к моделям (обязателен для PyAnnote). | Всегда устанавливайте в .env для gated-моделей. |
| `HF_HOME` | `/root/.cache/huggingface` | Путь к кэшу Hugging Face (монтируется как volume). | Если нужно изменить путь кэша для повторного использования. |
| `LANGUAGE` | `ru` | Код языка для распознавания (ISO-639-1, e.g., en, ru). | Для аудио на другом языке; влияет на точность. |
| `MODEL` | `large-v3` | Модель Whisper | Для кастомных моделей, e.g., `antony66/whisper-large-v3-russian` (после фиксов багов). |
| `DEVICE` | `cuda` | Устройство (`cuda` для GPU, `cpu` для CPU). | Если нет NVIDIA GPU или для тестирования. |
| `COMPUTE_TYPE` | `float16` | Точность вычислений (`float16`, `int8` для скорости на слабом железе). | `int8` для экономии VRAM, но с потерей точности. |
| `BATCH_SIZE` | `2` | Размер батча для обработки (выше = быстрее на GPU). | Увеличьте на мощном GPU (e.g., 4–8), уменьшите при OOM-ошибках. |
| `BEAM_SIZE` | `5` | Ширина beam search в декодировании. | В таблице ниже. |
| `TEMPERATURE` | `0` | Начальная температура декодирования (`0` = детерминистично). | В таблице ниже. |
| `TEMP_INC` | `0.2` | Инкремент температуры при fallback (если уверенность низкая, пробует выше температуры). | Для неоднозначного аудио; e.g., `0.2 0.4 0.6` для последовательных попыток. |
| `VAD` | `pyannote` | Метод VAD (`pyannote` для точного, `silero` для быстрого). | `silero` если PyAnnote не работает или для CPU. |
| `VAD_ONSET` | `0.30` | Порог начала речи (см. таблицу). | В таблице ниже. |
| `VAD_OFFSET` | `0.25` | Порог конца речи (см. таблицу). | В таблице ниже. |
| `MIN_SPK` | `1` | Минимальное число спикеров в диаризации (см. таблицу). | В таблице ниже. |
| `MAX_SPK` | `4` | Максимальное число спикеров (см. таблицу). | В таблице ниже. |
| `FORMAT` | `srt` | Формат вывода (`srt`, `vtt`, `txt`, `json`). | Для других нужд, e.g., `vtt` для веб-субтитров. |
| `HF_HUB_ENABLE_HF_TRANSFER` | `0` | Включает/выключает `hf_transfer` для скачивания (`0` = отключено для тишины в логах). | Включите (`1`) для ускорения скачивания больших моделей. |
| `LENGTH_PENALTY` | `1.1` (в run.sh) | Штраф за длину в beam search (`>1` поощряет длинные фразы). | Уменьшите (`<1`) для коротких ответов; влияет на coherentность. |


### Ключевые параметры для точности

Для точной транскрипции, особенно с шумным или многоспикерным аудио вроде интервью, настройте эти в `run.sh` или через env vars. Фокус на VAD для лучшего обнаружения сегментов и диаризации для разделения спикеров.

| Параметр | Диапазон/По умолчанию | За что отвечает | Эффект при уменьшении значения | Эффект при увеличении значения |
| :--- | :--- | :--- | :--- | :--- |
| `--vad_onset` | 0–1 (0.30) | Порог начала речи (логит > onset начинает сегмент). | Больше сегментов (ловит тихую речь, но включает больше шума/ложных срабатываний). | Меньше сегментов (пропускает тихую речь, снижает шум). |
| `--vad_offset` | 0–1 (0.25) | Порог конца речи (логит < offset заканчивает сегмент). | Короткие сегменты (больше прерываний, точные обрезы). | Длинные сегменты (меньше разрывов, но может слить спикеров/шум). |
| `--beam_size` | 1–10 (5) | Ширина поиска при декодировании Whisper (ищет больше гипотез). | Быстрее, но пропускает редкие слова/фразы. | Медленнее, но выше точность на сложном словаре (например, имена собственные на русском). |
| `--temperature` | 0–1 (0) | Случайность в декодировании; фолбэк-температуры (например, `0 0.2 0.4`) для неуверенности. | Более детерминистично (быстрее, но жестко). | Более разнообразные выводы (лучше на неоднозначном аудио, но медленнее). |
| `--min_speakers` / `--max_speakers` | ≥1 (1/4) | Диапазон кластеров спикеров в диаризации. | Меньше спикеров (сливает похожие голоса). | Больше спикеров (разделяет нюансы, но может переразделить). |

## HTTP API (FastAPI)

Для удалённой загрузки файла и получения транскрипта добавлен сервис `api` (FastAPI) с простой веб-страницей.

### Запуск API

```
docker compose up -d --build api
```

Откройте:

- Веб-страница загрузки: `http://localhost:8800/`
- Swagger-документация: `http://localhost:8800/docs`

Параметры распознавания можно менять «на горячую» на странице `http://localhost:8800/settings` — они сохраняются в `./config/settings.json` и применяются к новым транскрипциям. Значения по умолчанию берутся из переменных окружения контейнера (см. `docker-compose.yml`).

### Пример запроса CLI

```
curl -F "file=@audio/sample.mp3" \
     http://localhost:8800/api/transcribe

# Получить список задач
curl http://localhost:8800/api/jobs

# Скачать готовый результат
curl -OJ http://localhost:8800/download/<job_id>
```

### Реверс‑прокси через Nginx (скелет)

```
server {
    listen 80;
    server_name your.api.domain;

    location / {
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_pass http://127.0.0.1:8800;
    }
}
```

Документация FastAPI по установке, примерам и интерактивной документации: [fastapi.tiangolo.com](https://fastapi.tiangolo.com/)
